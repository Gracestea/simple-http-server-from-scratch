# Отчёт по проектной практике

## 1. Постановка задачи

Целью проектной практики являлась разработка программного компонента с нуля.

В качестве темы был выбран HTTP-сервер, реализованный на языке Python без использования сторонних веб-фреймворков.

Основные задачи проекта:

- изучить принципы работы HTTP;
- исследовать клиент-серверную архитектуру;
- реализовать работающий сервер;
- создать техническую документацию;
- подготовить репозиторий проекта.

---

## 2. Исследование технологии HTTP

Перед началом разработки были изучены:

- принцип работы HTTP-протокола;
- структура HTTP-запросов и ответов;
- TCP/IP соединения;
- работа сокетов;
- существующие HTTP-серверы.

Также были рассмотрены аналоги:

- Apache HTTP Server;
- Nginx;
- встроенный HTTP-сервер Python.

---

## 3. Проектирование приложения

На этапе проектирования была определена структура проекта:

```text
simple-http-server/
├── server.py
├── static/
├── logs/
└── docs/
```
UML-диаграмма отражает последовательность обработки HTTP-запроса сервером.
![alt text](<UML Activity Diagram.png>)

Сервер должен был:

- принимать подключения;
- обрабатывать GET-запросы;
- отдавать HTML-файлы;
- возвращать ошибку 404;
- вести журнал запросов.

---

## 4. Реализация HTTP-сервера

Для реализации использовался язык Python и модуль socket.

Были реализованы:

- серверный сокет;
- обработка подключений;
- разбор HTTP-запросов;
- генерация HTTP-ответов;
- работа со статическими файлами;
- обработка ошибок.

---

## 5. Тестирование

В процессе тестирования были успешно проверены:

- запуск сервера;
- подключение через браузер;
- отображение страниц;
- работа CSS;
- обработка ошибки 404;
- запись логов.

Также был протестирован переход между страницами.

---

## 6. Реализованные модификации

Дополнительно были реализованы:

- логирование запросов;
- поддержка CSS;
- отдельная директория static;
- обработка ошибки 404;
- несколько HTML-страниц;
- система хранения логов.

Данные функции расширили возможности базового HTTP-сервера.

---

## 7. Результаты работы

В результате проектной практики был создан минимально жизнеспособный HTTP-сервер.

Сервер способен:

- принимать HTTP-запросы;
- обрабатывать подключения;
- возвращать HTML-страницы;
- обрабатывать ошибки;
- сохранять логи запросов.
